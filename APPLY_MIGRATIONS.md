# Как применить миграции в Supabase

## Способ 1: Через SQL Editor (Рекомендуется) ⭐

### Шаг 1: Откройте SQL Editor
1. Перейдите на [Supabase Dashboard](https://app.supabase.com)
2. Выберите ваш проект `vgpsnbmypxxkzgwpcvwe`
3. В левом меню нажмите **SQL Editor**
4. Нажмите **New query**

### Шаг 2: Примените миграции по порядку

**⚠️ ВАЖНО: Выполняйте миграции строго по порядку!**

#### Миграция 1: Основная схема
1. Откройте файл: `supabase/migrations/20240101000000_create_sat_schema.sql`
2. Скопируйте **весь** содержимое файла
3. Вставьте в SQL Editor
4. Нажмите **Run** (или Ctrl+Enter)
5. Дождитесь сообщения "Success. No rows returned"

#### Миграция 2: Функции статистики
1. Откройте файл: `supabase/migrations/20240101000001_create_stats_update_function.sql`
2. Скопируйте содержимое
3. Вставьте в SQL Editor (можно в том же окне, после предыдущей)
4. Нажмите **Run**

#### Миграция 3: Подписки (опционально)
1. Откройте файл: `supabase/migrations/20240101000002_create_subscriptions.sql`
2. Скопируйте содержимое
3. Вставьте и выполните

#### Миграция 4: Платежи (опционально)
1. Откройте файл: `supabase/migrations/20240101000003_create_pending_payments.sql`
2. Скопируйте содержимое
3. Вставьте и выполните

#### Миграция 5: Пользовательские вопросы
1. Откройте файл: `supabase/migrations/create_user_questions.sql`
2. Скопируйте содержимое
3. Вставьте и выполните

### Шаг 3: Проверка
1. Перейдите в **Table Editor** в левом меню
2. Вы должны увидеть созданные таблицы:
   - `questions`
   - `lessons`
   - `flashcards`
   - `user_progress`
   - `user_bookmarks`
   - `user_stats`
   - `user_questions` (если применили последнюю миграцию)
   - И другие...

## Способ 2: Через Supabase CLI

Если у вас установлен Supabase CLI:

```bash
# 1. Установите Supabase CLI (если еще не установлен)
npm install -g supabase

# 2. Войдите в Supabase
supabase login

# 3. Свяжите проект с локальными миграциями
supabase link --project-ref vgpsnbmypxxkzgwpcvwe

# 4. Примените все миграции
supabase db push
```

## Проверка успешного применения

После применения миграций проверьте:

1. **Таблицы созданы:**
   - Table Editor → должны быть все таблицы

2. **RLS включен:**
   - Table Editor → выберите таблицу → Settings → Row Level Security должен быть включен

3. **Политики созданы:**
   - Authentication → Policies → должны быть политики для user_* таблиц

## Решение проблем

### Ошибка "relation already exists"
- Таблица уже существует, это нормально
- Продолжайте с следующей миграцией

### Ошибка "permission denied"
- Убедитесь, что вы вошли в правильный проект
- Проверьте права доступа к проекту

### Ошибка "syntax error"
- Проверьте, что скопировали весь файл
- Убедитесь, что нет лишних символов

### Таблицы не видны
- Обновите страницу Table Editor
- Проверьте, что миграции выполнились успешно (зеленая галочка в SQL Editor)
