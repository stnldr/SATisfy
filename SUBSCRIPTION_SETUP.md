# Система подписок - Руководство по настройке

## ✅ Что было создано

Система подписок для StudySAT полностью реализована и готова к использованию.

## 📊 База данных

### Миграция создана:
- `supabase/migrations/20240101000002_create_subscriptions.sql`
  - Таблицы: `subscription_plans`, `user_subscriptions`, `subscription_history`
  - Функции для проверки доступа к функциям
  - RLS политики для безопасности

### Планы подписки:

1. **Бесплатный (Free)**
   - 10 вопросов в день
   - Ограниченный доступ к урокам
   - Базовые флеш-карты
   - Базовая статистика
   - Цена: $0

2. **Базовый (Basic)** - $9.99/месяц или $99.99/год
   - Неограниченные вопросы
   - Все уроки
   - Все флеш-карты
   - Расширенная статистика

3. **Премиум (Premium)** - $19.99/месяц или $199.99/год
   - Все возможности Базового
   - Продвинутая аналитика
   - Приоритетная поддержка
   - Эксклюзивные материалы

4. **Профессиональный (Pro)** - $39.99/месяц или $399.99/год
   - Все возможности Премиум
   - Персональный тренер
   - Кастомные планы обучения
   - API доступ

## 🎨 Страницы

### 1. `/pricing` - Выбор плана подписки
- Отображение всех доступных планов
- Сравнение функций
- Кнопки оформления подписки
- FAQ секция

### 2. `/subscription` - Управление подпиской
- Просмотр текущей подписки
- Детали плана
- Дата следующего продления
- Отмена подписки
- История подписки

## 🔐 Защита функций

### Компоненты:

1. **`SubscriptionGate`** - Обертка для защиты премиум функций
   ```tsx
   <SubscriptionGate feature="unlimited_questions">
     <PremiumComponent />
   </SubscriptionGate>
   ```

2. **`QuestionLimitGate`** - Автоматическая проверка лимита вопросов
   - Предупреждение при приближении к лимиту
   - Блокировка при достижении лимита
   - Призыв к оформлению подписки

### Использование:

В `Practice.tsx` добавлена автоматическая проверка лимита:
- Бесплатные пользователи: 10 вопросов в день
- Премиум пользователи: без ограничений

## 📝 Интеграция

### Навигация:
- Кнопка "Подписка" в меню пользователя
- Кнопка "Тарифы" в шапке для неавторизованных пользователей
- Ссылки на `/pricing` и `/subscription`

### Хуки:

- `useSubscriptionPlans()` - Получить все планы
- `useUserSubscription()` - Получить подписку пользователя
- `useCreateSubscription()` - Оформить подписку
- `useCancelSubscription()` - Отменить подписку
- `useFeatureAccess()` - Проверить доступ к функции
- `useTodaysQuestionCount()` - Количество вопросов сегодня

## ⚙️ Настройка

### 1. Запустить миграцию

Выполните SQL миграцию в Supabase SQL Editor:

```sql
-- Запустите файл: supabase/migrations/20240101000002_create_subscriptions.sql
```

Это создаст:
- Все необходимые таблицы
- Планы подписки (автоматически вставлены)
- Функции проверки доступа
- RLS политики

### 2. Проверить данные

После миграции проверьте, что планы созданы:

```sql
SELECT * FROM subscription_plans;
```

Должно быть 4 плана: free, basic, premium, pro

### 3. Тестирование

1. Откройте `/pricing` - должны отображаться все планы
2. Попробуйте оформить подписку (пока без реальной оплаты)
3. Проверьте `/subscription` - должна отображаться информация о подписке
4. На странице Practice - должно работать ограничение для бесплатных пользователей

## 💳 Интеграция платежей (Stripe)

**ВНИМАНИЕ**: Текущая реализация создает подписку в базе данных, но НЕ интегрирована с реальной платежной системой.

### Для полной интеграции с Stripe:

1. Установите Stripe:
   ```bash
   npm install @stripe/stripe-js
   ```

2. Создайте Stripe Webhook для обработки событий
3. Обновите `createSubscription()` в `src/lib/supabase/subscriptions.ts`
4. Добавьте обработку платежей в Supabase Edge Functions

### Временное решение:

Сейчас подписка создается напрямую в базе данных для тестирования. В production нужно:
- Добавить проверку оплаты через Stripe
- Использовать Stripe Checkout для оплаты
- Обрабатывать события через webhooks

## 🔧 Настройка лимитов

Лимиты для бесплатного плана можно изменить в:
1. Миграции SQL (при создании плана)
2. `src/components/QuestionLimitGate.tsx` (константа `maxQuestions`)

## 📈 Функции доступа

Проверка доступа к функциям работает через функцию `has_subscription_access()`:

- `unlimited_questions` - доступно для basic, premium, pro
- `advanced_analytics` - доступно для premium, pro
- `priority_support` - доступно для premium, pro
- `all_lessons` - доступно для basic, premium, pro
- `all_flashcards` - доступно для basic, premium, pro
