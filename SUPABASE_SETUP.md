# Инструкция по подключению Supabase

## Шаг 1: Создание проекта в Supabase

1. Перейдите на [https://supabase.com](https://supabase.com)
2. Войдите или зарегистрируйтесь
3. Нажмите "New Project"
4. Заполните данные:
   - **Name**: StudySAT (или любое другое имя)
   - **Database Password**: создайте надежный пароль (сохраните его!)
   - **Region**: выберите ближайший регион
5. Нажмите "Create new project"
6. Дождитесь создания проекта (обычно 1-2 минуты)

## Шаг 2: Получение ключей API

1. В вашем проекте Supabase перейдите в **Settings** → **API**
2. Найдите следующие значения:
   - **Project URL** (например: `https://xxxxx.supabase.co`)
   - **anon public** ключ (начинается с `eyJ...`)

## Шаг 3: Настройка переменных окружения

1. Создайте файл `.env` в корне проекта (скопируйте `.env.example`)
2. Заполните переменные:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-anon-key-here
```

**Важно**: 
- Не коммитьте файл `.env` в Git (он уже в `.gitignore`)
- Используйте `.env.example` как шаблон

## Шаг 4: Создание таблиц в базе данных

### Вариант A: Через SQL Editor (рекомендуется)

1. В Supabase Dashboard перейдите в **SQL Editor**
2. Создайте новый запрос
3. Выполните миграции **в следующем порядке**:

   **a) Основная схема:**
   - Скопируйте содержимое `supabase/migrations/20240101000000_create_sat_schema.sql`
   - Выполните SQL запрос

   **b) Функции статистики:**
   - Скопируйте содержимое `supabase/migrations/20240101000001_create_stats_update_function.sql`
   - Выполните SQL запрос

   **c) Подписки (если нужны):**
   - Скопируйте содержимое `supabase/migrations/20240101000002_create_subscriptions.sql`
   - Выполните SQL запрос

   **d) Платежи (если нужны):**
   - Скопируйте содержимое `supabase/migrations/20240101000003_create_pending_payments.sql`
   - Выполните SQL запрос

   **e) Пользовательские вопросы:**
   - Скопируйте содержимое `supabase/migrations/create_user_questions.sql`
   - Выполните SQL запрос

**Важно**: Выполняйте миграции по порядку, так как они зависят друг от друга!

### Вариант B: Через Supabase CLI

Если у вас установлен Supabase CLI:

```bash
# Установка Supabase CLI (если еще не установлен)
npm install -g supabase

# Логин в Supabase
supabase login

# Связывание проекта
supabase link --project-ref your-project-ref

# Применение миграций
supabase db push
```

## Шаг 5: Настройка Row Level Security (RLS)

RLS уже настроен в миграциях, но убедитесь, что:

1. Перейдите в **Authentication** → **Policies**
2. Проверьте, что для таблицы `user_questions` включен RLS
3. Политики должны быть созданы автоматически через миграцию

## Шаг 6: Проверка подключения

1. Запустите проект:
```bash
npm run dev
```

2. Откройте консоль браузера (F12)
3. Проверьте, что нет ошибок подключения к Supabase
4. Попробуйте зарегистрироваться/войти

## Шаг 7: Заполнение начальных данных (опционально)

Если нужно заполнить базу начальными вопросами:

1. Перейдите в **Table Editor** в Supabase Dashboard
2. Выберите таблицу `questions`
3. Добавьте вопросы вручную или используйте SQL:

```sql
INSERT INTO questions (section, difficulty, question, options, answer, explanation)
VALUES 
  ('Reading', 'easy', 'Ваш вопрос?', ARRAY['Вариант 1', 'Вариант 2', 'Вариант 3', 'Вариант 4'], 0, 'Объяснение');
```

## Структура таблиц

### Основные таблицы:

1. **questions** - стандартные вопросы SAT
2. **lessons** - уроки
3. **flashcards** - флеш-карты
4. **user_questions** - пользовательские вопросы (создается миграцией)
5. **user_progress** - прогресс пользователя
6. **user_bookmarks** - закладки пользователя
7. **user_stats** - статистика пользователя

## Проверка работы

После настройки проверьте:

1. ✅ Регистрация/вход работают
2. ✅ Вопросы загружаются на странице Practice
3. ✅ Можно создавать свои вопросы на странице Questions
4. ✅ Прогресс сохраняется
5. ✅ Закладки работают

## Решение проблем

### Ошибка "Invalid API key"
- Проверьте, что скопировали правильный `anon` ключ
- Убедитесь, что файл `.env` находится в корне проекта
- Перезапустите dev сервер после изменения `.env`

### Ошибка "relation does not exist"
- Убедитесь, что выполнили все миграции
- Проверьте, что таблицы созданы в Supabase Dashboard → Table Editor

### Ошибка "permission denied"
- Проверьте, что RLS политики настроены правильно
- Убедитесь, что пользователь авторизован

### Данные не сохраняются
- Проверьте консоль браузера на ошибки
- Убедитесь, что пользователь залогинен
- Проверьте RLS политики в Supabase Dashboard

## Дополнительная настройка

### Настройка Email для аутентификации

1. Перейдите в **Authentication** → **Providers**
2. Настройте Email provider
3. Добавьте SMTP настройки для отправки писем (опционально)

### Настройка домена

1. Перейдите в **Settings** → **API**
2. Добавьте ваш домен в **Allowed URLs** (для production)
